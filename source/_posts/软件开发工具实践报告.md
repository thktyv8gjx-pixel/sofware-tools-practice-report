## 实验二：代码编辑与管理

### 实验目的

本实验旨在掌握 Visual Studio Code 编辑器的基本使用方法，学习 Markdown 常用语法，并使用 Git 对项目文件进行版本控制。通过多次修改、暂存和提交，理解工作区、暂存区和本地仓库之间的关系。

### Markdown 编辑与渲染

本实验使用 Markdown 编写实验报告。Markdown 具有语法简洁、可读性强和便于版本管理等特点。

| 语法功能 | Markdown 写法 | 显示效果 |
| --- | --- | --- |
| 一级标题 | `# 标题` | 一级标题 |
| 二级标题 | `## 标题` | 二级标题 |
| 加粗 | `**文字**` | **文字** |
| 斜体 | `*文字*` | *文字* |
| 无序列表 | `- 内容` | 项目列表 |
| 行内代码 | `` `代码` `` | `代码` |

### Git 核心概念

1. **工作区**：当前正在编辑的项目文件；
2. **暂存区**：准备在下一次提交中保存的修改；
3. **本地仓库**：保存项目完整提交历史的位置；
4. **提交**：为当前暂存内容创建一个版本快照。

Git 的基本工作流程为：

> 修改文件 → 暂存更改 → 提交版本
